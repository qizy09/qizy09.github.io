Gesture detectors
There are multiple ways to achieve gesture detecting. Android gesture detectors.
The basic structure of implementing the gesture detector is to take advantage of the OnTouchListener. In your Activity class you implement the android.view.View.OnTouchListener which makes you implement the onTouch(...) method. This way the Android system is able to inform the class any android.view.MotionEvent is happening.
Android has a ScaleGestureDetector since API level 8 (Android 2.2), and therefore Android devices with multitouch screens appeared. To initialize the ScaleGestureDetector, we want to start with initialization ScaleListener, so as to know when a scale event has happened and what scaling information is available.
There is an opensource package providing the interfaces we could take advantage of, named Android Gesture Detectors Framework, including RotateGestureDetector, MoveGestureDetector, ScaleGestureDetector, etc. which is useful for this project.
Play Slides
In order to setup an activity that plays slides inside the app, I considered multiple potential solutions to achieve this.
Playing slides in Microsoft Powerpoint format(ppt/pptx) is definitley desired, since it's most widely used. However, it seems hard to find any official tutorials to implement or open source support to achieve this.
Another alternative solution is to play html5 slides. Google provided an HTML5 Slide Template in io 2012, and it's still under maintainance. So another solution is to implement an activity subclassing Webview and embed the html5 slides inside.
Android provoides a WebView class, which is a View that displays web pages, useing the WebKit rendering engine to display web pages and includes methods to navigate forward and backward through a history, zoom in and out, perform text searches and more.
And there are definitely potential extension of this alternative solution, i.e. to extend the html5 with some plugins/lbraries that could play Microsoft Powerpoint/ Google Slides using html5. I'll look at that part later.
Notes and sharing
This app use Camera access permission to take notes, and share with team members using Bluetooth.
The Android platform includes support for the Bluetooth network stack, which allows a device to wirelessly exchange data with other Bluetooth devices, with the application framework providing access to the Bluetooth functionality through the Android Bluetooth APIs.
Google Drive Sharing
With Internet access, this app use the Drive REST API web API to send it to cloud and share with others.
